# 03-simple-skill

## 问题分析：Skill 未能自动触发

当用户输入 `hello` 时，`say-hello` 技能没有被激活，原因如下：

### 1. 技能需要手动加载

系统通过 `available_skills` 列出所有可用的技能，并提示模型：

> "Use the skill tool to load a skill when a task matches its description."

技能**不会自动匹配触发**，必须由模型主动调用 `skill` 工具来加载。模型需要根据 `description` 判断是否应该加载某个技能。

### 2. 模型未识别出加载时机

用户说 `hello` 时，模型将其当作普通打招呼直接回复，没有意识到应先调用 `skill` 工具加载 `say-hello` 再响应。

### 3. Skill Body 指令缺少回溯检查

原 SKILL.md body 内容为：

```
请在收到用户提示词包含 hello 的时候回复"欢迎光临"。
```

此指令暗示模型"等待未来消息"，但技能加载时用户的 `hello` 消息**已经发生**。指令应明确要求模型**检查当前对话历史**。

### 4. 缺少未命中时的兜底行为

当用户消息不含 `hello` 时，没有说明该怎么做，容易导致误触发。

---

## 优化方案

### 优化后的 `SKILL.md`

```markdown
---
name: say-hello
description: 当用户消息包含 "hello"（不区分大小写）时加载此技能，以中文回复。
---

检查当前对话中用户最近的消息是否包含 "hello"（不区分大小写）：
- 如果是，回复"欢迎光临"
- 如果否，保持沉默，不做任何回复
```

### 优化要点

| 问题 | 优化 |
|------|------|
| 描述不够精确 | 补充 `不区分大小写`，避免漏匹配 `Hello`/`HELLO` |
| 指令面向未来消息 | 改为 `检查当前对话中用户最近的消息`，明确回溯已有消息 |
| 缺少未命中行为 | 补充 `如果否，保持沉默`，防止无关场景误触发 |
